# Data Dictionary

## Raw Data (`data/raw/`)

Each CSV file (`matches_2020_2021.csv`, etc.) contains one season of match records.

### Schema

| Column | Type | Description |
|--------|------|-------------|
| `match_id` | int | Unique match identifier (from football-data.org) |
| `date` | datetime | UTC match kickoff time (ISO 8601) |
| `season` | int | Season year (e.g., 2024 for 2024-25 season) |
| `matchday` | int | Matchday number (1-38 in Premier League) |
| `home_team` | str | Home team name |
| `away_team` | str | Away team name |
| `home_score` | int | Final home team score (null if not played yet) |
| `away_score` | int | Final away team score (null if not played yet) |
| `status` | str | Match status: "SCHEDULED", "LIVE", "FINISHED", "POSTPONED", "CANCELLED" |
| `attendance` | int | Official attendance or estimated (~70% of stadium capacity if unavailable) |

### Data Quality Notes

- **Attendance**: Free tier of football-data.org doesn't guarantee actual attendance for all matches. For missing values, we estimate as 70% of stadium capacity (documented as a deliberate design choice).
- **Dates**: All times are in UTC. Adjust for your timezone as needed.
- **Duplicate handling**: Raw CSVs may contain duplicate match_ids if re-downloaded; `pipeline/refresh.py` deduplicates on upsert.

## Processed Data (`data/processed/`)

Generated by the pipeline—do not edit manually.

### `features.parquet`

Feature-engineered dataset used for training. One row per completed match.

**Columns** (14 features + metadata):

| Feature | Type | Range | Description |
|---------|------|-------|-------------|
| `match_id` | int | N/A | Reference to raw match |
| `date` | datetime | N/A | Match date |
| `home_team` | str | N/A | Home team (for reference) |
| `away_team` | str | N/A | Away team (for reference) |
| `season` | int | 2020-2024 | Season year |
| `matchday` | int | 1-38 | Matchday in season |
| `home_win_rate_last5` | float | 0.0-1.0 | Home team's win rate in last 5 home games |
| `away_win_rate_last5` | float | 0.0-1.0 | Away team's win rate in last 5 away games |
| `home_goals_scored_avg` | float | 0.5-5.0 | Home team's avg goals in last 5 games (any role) |
| `away_goals_conceded_avg` | float | 0.5-5.0 | Away team's avg goals conceded (when home) last 5 games |
| `is_rivalry` | bool | True/False | True if top-20 historical rivalry |
| `days_since_last_home_game` | int | 1-30 | Rest days for home team |
| `season_progress` | float | 0.0-1.0 | matchday / 38 (where in season) |
| `home_league_position` | int | 1-20 | Estimated home team position |
| `away_league_position` | int | 1-20 | Estimated away team position |
| `position_gap` | int | 0-20 | |home_pos - away_pos| |
| `is_weekend` | bool | True/False | True if Saturday or Sunday |
| `month` | int | 1-12 | Month of match |
| `stadium_capacity` | int | 11k-74k | Home stadium capacity (from settings) |
| `attendance_pct` | float | 0.0-1.0 | **Target**: attendance / stadium_capacity |

### `model.pkl`

Trained LightGBM regressor (joblib format). Produced by `pipeline/train.py`.

Predicts `attendance_pct` given 14 feature columns.

### `forecasts.parquet`

Upcoming fixture predictions. Produced by `pipeline/forecast.py`.

| Column | Type | Description |
|--------|------|-------------|
| `match_id` | int | Upcoming fixture ID |
| `date` | datetime | Predicted match date |
| `home_team` | str | Home team |
| `away_team` | str | Away team |
| `matchday` | int | Matchday (1-38) |
| `predicted_attendance` | float | Point estimate (absolute count) |
| `lower_bound` | float | 80% PI lower (absolute count) |
| `upper_bound` | float | 80% PI upper (absolute count) |
| `predicted_attendance_pct` | float | Point estimate as % of capacity |
| `model_version` | str | Model identifier (e.g., "lgb_v1") |
| `forecast_timestamp` | str | ISO 8601 timestamp when forecast was generated |

### `backtest_results.csv`

Rolling-origin backtest performance. Produced by `pipeline/train.py`.

| Column | Type | Description |
|--------|------|-------------|
| `train_seasons` | str | Seasons used for training (e.g., "[2020, 2021]") |
| `test_season` | int | Season evaluated (e.g., 2022) |
| `test_size` | int | Number of matches in test set |
| `mape` | float | Mean Absolute Percentage Error on test set |

## Updating Data

To refresh data with the latest matches:

```bash
python -m pipeline.refresh   # Incremental: last 7 days + next 14 days
```

To backfill all historical seasons (only needed once):

```bash
python -m pipeline.backfill
```

The daily GitHub Actions workflow (`data-refresh.yml`) runs this automatically at 7am UTC.
